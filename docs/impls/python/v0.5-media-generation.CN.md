# Python v0.5 Media Generation 方案

文档类型：实现方案与实现记录
日期：2026-05-29
最近更新：2026-05-29
引入需求：[REQ-2026-05-python-reference-implementation-roadmap.CN.md](../../requirements/REQ-2026-05-python-reference-implementation-roadmap.CN.md)

## 背景

Python v0.4 已完成 Volcengine adapter MVP，使用 Ark SDK 原生 surface 覆盖 Responses generation/streaming、Files API、多模态 embedding、function tools、reasoning、structured output、provider-native replay 与 remote context fallback。

v0.5 的目标是进入 media generation API family，并用 OpenAI 与 Volcengine 两个 provider 共同校准 core model。图片生成同时覆盖 OpenAI Images API 与 Volcengine Ark Images API；视频生成先覆盖 Volcengine Ark content generation task。

历史 evolution plan 中把 hosted tools 与 media generation 放在同一阶段；这部分 hosted tools 规划已经过期，v0.5 暂不实现 provider-hosted tools、Remote MCP、Web Search helper、Image Process helper 或 Knowledge Search helper。

## 设计原则

### Media Generation 独立于 Generation

图片/视频生成不复用 `GenerationRequest`。它们使用 media generation family 的 request、artifact、stream event 和 async task 模型，避免把对话上下文、function calling、reasoning 与媒体产物生命周期混在一起。

### Provider Native SDK-only

Volcengine v0.5 继续仅使用 Ark SDK 原生接口：

- 图片生成：`Ark.images.generate` / `AsyncArk.images.generate`。
- 图片流式生成：`Ark.images.generate(stream=True)` / async streaming。
- 视频任务创建：`Ark.content_generation.tasks.create`。
- 视频任务查询：`Ark.content_generation.tasks.get`。

不使用火山方舟 OpenAI-compatible SDK surface，不把 OpenAI SDK 配置为火山方舟 base URL，也不引入 direct HTTP fallback。

OpenAI v0.5 仅考虑直接调用 Images API：

- 图片生成：`OpenAI.images.generate` / `AsyncOpenAI.images.generate`。
- 带参考图的图片生成：`OpenAI.images.edit` / `AsyncOpenAI.images.edit`。
- 图片流式生成：`OpenAI.images.generate(stream=True)` 或 `OpenAI.images.edit(stream=True)` / async streaming。

不考虑通过 GPT-5.4+ 等对话/Responses 模型调用内置 image generation tool 的间接路径，不把 Responses API hosted tool 事件映射为 media generation。

### Hosted Tools 暂不进入范围

`ImageGenerationRequest.tools` 已从 core 中删除。v0.5 不新增 `VideoGenerationRequest.tools`，也不把 provider-hosted/remote tools 映射为 `ToolSpec`。

原因：

- `ToolSpec` 表达 user-executed function/custom tools，执行责任属于用户代码。
- OpenAI Responses 内置图片工具、Volcengine 图片/视频生成中的联网搜索、图像处理、知识检索或 MCP 能力都是 provider-native hosted capability，执行责任、权限、计费、可观测性和生命周期均不同。
- 当前阶段没有成熟的跨 provider 执行责任模型。若用户必须使用 provider 原生开关，应通过 `provider_options` 显式传递。

## Scope

### Image Generation

OpenAI 与 Volcengine 共同支持的通用能力：

- text-to-image。
- reference image / image-to-image。
- output format。
- response format。
- count。
- stream events。
- partial image/artifact events。
- usage mapping。

OpenAI 与 Volcengine 的 API 形状不同：Volcengine 用一个 `images.generate` surface 同时承载纯文本生成与参考图生成；OpenAI 在 Images API 内拆分为 `images.generate` 与 `images.edit`。aiflect 不把这种拆分暴露给用户，adapter 根据 `ImageGenerationRequest.input_items` 是否存在参考图自动选择 OpenAI `generate` 或 `edit`。

### Video Generation

Volcengine 支持：

- create video generation task。
- retrieve video generation task。
- wait/poll helper。
- task status mapping。
- artifact mapping。
- error mapping。
- text/image/video/audio reference content 的最小映射。

### Capabilities

OpenAI adapter capability 应声明：

- `MediaGenerationCapability.image_generation=True`。
- `MediaGenerationCapability.video_generation=False`。
- `MediaGenerationCapability.streaming=True`，表示 Images API stream/partial images。
- `MediaGenerationCapability.async_task=False`。

Volcengine adapter capability 应声明：

- `MediaGenerationCapability.image_generation=True`。
- `MediaGenerationCapability.video_generation=True`。
- `MediaGenerationCapability.streaming=True`，表示图片生成流式事件。
- `MediaGenerationCapability.async_task=True`，表示视频生成任务模型。

model capability 仍默认 unknown，允许用户 overrides。

## Non-scope

v0.5 不做：

- hosted tools / Remote MCP / Web Search helper / Image Process helper / Knowledge Search helper。
- media hosted tools 的稳定 core 抽象。
- 自动执行任何工具。
- 自动下载、保存、转码生成的媒体文件。
- 隐式上传本地参考文件。
- OpenAI image variation API。
- 通过 OpenAI Responses API 内置 image generation tool 间接生成图片。
- 图片/视频编辑高级参数标准化。
- direct HTTP fallback。
- provider conversation 或跨 provider replay。

## Core 模型调整

### ImageGenerationRequest

从 OpenAI Images API 与 Volcengine Ark Images API 的交集中保留：

```text
model
prompt
input_items
quality
background
output_format
response_format
count
watermark
stream_options
provider_options
```

删除：

```text
tools
```

字段语义：

- `model`：provider model id。
- `prompt`：图片生成提示词。
- `input_items`：参考图片、文件等上下文；OpenAI adapter 根据该字段自动选择 `images.generate` 或 `images.edit`，Volcengine adapter 映射为 Ark `images.generate` 的 reference image 参数。
- `quality`：图片质量。OpenAI 可映射到 `quality`；Volcengine Ark Images API 当前没有等价字段，adapter 忽略该字段。
- `background`：背景模式。OpenAI 可映射到 `background`；Volcengine Ark Images API 当前没有等价字段，adapter 忽略该字段，并在 `MediaGenerationCapability.image_background_control` 中声明不支持。
- `output_format`：输出格式，例如 `png`、`jpeg`、`webp`。
- `response_format`：返回形态，例如 `url` 或 `b64_json`。
- `count`：生成数量；OpenAI 映射 `n`，Volcengine 映射 Ark `sequential_image_generation_options.max_images`。
- `watermark`：是否要求 provider 为生成图片添加 AI 水印，默认 `True`。若 provider 或模型没有可控水印能力，adapter 忽略该字段。
- `provider_options`：OpenAI `moderation`、`output_compression`、`partial_images`、`user` 或 Volcengine `sequential_image_generation` 等原生参数。

`tools` 删除后，图片生成不会暴露普通 function tool 编程模型。Provider 原生媒体生成选项通过 `provider_options` 传递。

`size` 不保留为 normalized 字段。OpenAI 与 Volcengine 在图片生成时对分辨率参数的取值要求不同，且多数图片生成模型具备从 prompt 中感知目标分辨率、长宽比和构图规格的能力。v0.5 当前采用 auto 模式：默认不向 provider 请求层发送分辨率参数，由模型和 provider 默认策略自行决定。若用户确实需要 provider-native 分辨率控制，应通过 `provider_options` 显式传递对应 provider 原生字段。

### VideoGenerationRequest

新增 `VideoGenerationRequest`：

```text
model
prompt
input_items
duration_seconds
ratio
resolution
generate_audio
watermark
stream_options
provider_options
```

不包含：

```text
tools
```

字段语义：

- `model`：provider model id。
- `prompt`：视频生成提示词。
- `input_items`：参考图片、视频、音频或文件。
- `duration_seconds`：目标时长。
- `ratio`：画幅比例，例如 `"16:9"`。
- `resolution`：输出规格，若 provider 支持则映射到原生字段。
- `generate_audio`：是否生成音频。
- `watermark`：是否要求 provider 为生成视频添加 AI 水印，默认 `True`。若 provider 或模型没有可控水印能力，adapter 忽略该字段。
- `provider_options`：seed、advanced controls 等 provider-native 参数。

### MediaArtifact

继续表达图片/视频产物：

```text
kind
url
data
file_id
mime_type
format
width
height
duration_seconds
provider
metadata
raw
```

### MediaGenerationTask

继续表达视频异步任务：

```text
id
provider
model
status
artifacts
error
created_at
updated_at
metadata
raw
```

任务状态映射：

```text
queued -> TaskStatus.QUEUED
running -> TaskStatus.RUNNING
succeeded -> TaskStatus.COMPLETED
completed -> TaskStatus.COMPLETED
failed -> TaskStatus.FAILED
canceled/cancelled -> TaskStatus.CANCELED
expired -> TaskStatus.EXPIRED
unknown/other -> TaskStatus.UNKNOWN
```

## Provider Client API

OpenAI 与 Volcengine 都实现统一图片生成方法：

```text
generate_image(...)
agenerate_image(...)
stream_generate_image(...)
astream_generate_image(...)
```

Volcengine 额外实现视频生成任务方法：

```text
create_video_generation_task(...)
acreate_video_generation_task(...)
get_video_generation_task(...)
aget_video_generation_task(...)
wait_for_video_generation_task(...)
await_video_generation_task(...)
```

OpenAI video generation 在 v0.5 中表达为 unsupported capability；如果调用相应方法，应抛 `UnsupportedCapabilityError`，或者不在 OpenAIClient 暴露视频方法。为了减少 public surface 噪声，v0.5 建议 OpenAIClient 只新增图片方法。

### OpenAI Image

同步：

```python
response = client.generate_image(
    model="gpt-image-1",
    prompt="a cinematic product photo",
    input_items=[...],
    quality="high",
    output_format="png",
    response_format="b64_json",
    provider_options={"partial_images": 2},
)

for event in client.stream_generate_image(...):
    ...
```

异步：

```python
response = await client.agenerate_image(...)

async for event in client.astream_generate_image(...):
    ...
```

### Volcengine Image

同步：

```python
response = client.generate_image(
    model="doubao-seedream-5-0-260128",
    prompt="a cinematic product photo",
    output_format="png",
    response_format="url",
    watermark=False,
)

for event in client.stream_generate_image(...):
    ...
```

异步：

```python
response = await client.agenerate_image(...)

async for event in client.astream_generate_image(...):
    ...
```

方法签名建议：

```text
generate_image(model, prompt, *, input_items=(), quality=None,
               background=None, output_format=None,
               response_format=None, count=None, watermark=True, stream_options=None,
               provider_options=None) -> ImageGenerationResponse

stream_generate_image(...) -> Iterator[ImageGenerationStreamEvent]
```

### Video

同步：

```python
from whero.aiflect import AudioPart, ImagePart, MessageItem, VideoPart

task = client.create_video_generation_task(
    model="doubao-seedance-2-0-260128",
    prompt="a short cinematic clip",
    input_items=[
        MessageItem.user([
            ImagePart(
                url="https://example.test/first-frame.png",
                metadata={"role": "first_frame"},
            ),
            VideoPart(url="https://example.test/motion-reference.mp4", fps=2.0),
            AudioPart(url="https://example.test/music.mp3"),
        ])
    ],
    duration_seconds=8,
    ratio="16:9",
    generate_audio=True,
    watermark=False,
)

task = client.get_video_generation_task(task.id)
task = client.wait_for_video_generation_task(task.id)
```

异步：

```python
task = await client.acreate_video_generation_task(...)
task = await client.aget_video_generation_task(task.id)
task = await client.await_video_generation_task(task.id)
```

方法命名采用 `get_video_generation_task`，因为它直接对应 Ark SDK `tasks.get`，语义是查询已创建任务，不重新生成。

视频生成的 `input_items` 是多模态 reference content，不是对话历史。Volcengine adapter 将 `TextPart`、`ImagePart`、`VideoPart`、`AudioPart` 和带 media type 的 `FilePart` 映射为 Ark Content Generation task 的 `content` 条目；`metadata["role"]` 可用于显式传递 provider-native role。

## OpenAI 映射设计

### Image Request

`ImageGenerationRequest` 到 OpenAI Images API：

```text
request.model -> model
request.prompt -> prompt
request.input_items empty -> images.generate
request.input_items with reference image(s) -> images.edit
request.quality -> quality
request.background -> background
request.output_format -> output_format
request.response_format -> response_format
request.count -> n
stream=True -> stream=True
provider_options -> top-level OpenAI image params
```

`provider_options` 推荐承载：

- `moderation`
- `output_compression`
- `partial_images`
- `user`

v0.5 不把 OpenAI `images.create_variation` 纳入通用面；参考图生成通过 `images.edit` 由 adapter 自动承接。

OpenAI route 选择：

```text
input_items == () -> client.images.generate(...)
input_items contains supported reference images -> client.images.edit(...)
```

OpenAI reference input mapping：

- `ImagePart(data=...)` -> `image` file/bytes input for `images.edit`。
- `ImagePart(url=...)` 不会被隐式下载，当前实现抛 `UnsupportedCapabilityError`。
- 多张 `ImagePart(data=...)` 参考图映射为 `image` 数组；单张参考图映射为单个 file/bytes input。
- `FilePart(...)` 不映射到 OpenAI image edit，当前实现抛 `UnsupportedCapabilityError`。

OpenAI `mask`、`input_fidelity` 等 edit-only 字段暂不进入 core，通过 `provider_options` 传递。

### Image Response

OpenAI image generate/edit response 到 `ImageGenerationResponse`：

```text
response.data[*].url -> MediaArtifact(kind=image, url=...)
response.data[*].b64_json -> MediaArtifact(kind=image, data=...)
response.data[*].revised_prompt -> artifact.metadata["revised_prompt"]
response.usage -> Usage
raw -> raw
```

### Image Stream

OpenAI image stream event 到 `ImageGenerationStreamEvent`：

```text
image_generation.partial_image or equivalent -> partial image event
image_generation.completed or final image item -> completed/artifact event
error event -> error event
unknown -> raw passthrough
```

具体 event type 以 OpenAI Python SDK `images.generate(stream=True)` / `images.edit(stream=True)` 实际对象为准，用 fixture test 锁定。实现不复用 Responses API image_generation tool stream event。

## Volcengine 映射设计

### Image Request

```text
request.model -> model
request.prompt -> prompt
request.quality -> ignored, Ark Images API has no equivalent field
request.background -> ignored, Ark Images API has no equivalent field
request.output_format -> output_format
request.response_format -> response_format
request.count -> sequential_image_generation_options.max_images
request.input_items -> image/reference image inputs
stream=True -> stream=True
provider_options -> top-level Ark params
```

火山方舟图片生成 API reference 显示 `size`、`sequential_image_generation`、`sequential_image_generation_options`、`tools`、`guidance_scale`、`optimize_prompt_options` 等均属于 provider-native 参数。v0.5 只把具备跨 provider 语义且当前实现稳定的字段放入 core；其余字段通过 `provider_options` 显式传递。Hosted tools 仍不进入 `ToolSpec`。

`ImagePart` 映射：

- URL -> Ark `image` URL。
- data/base64 -> Ark `image` data URL。
- 当前实现只接受 `ImagePart(url=...)` / `ImagePart(data=...)`；`FilePart(...)` 与其他 part 会抛 `InvalidItemError`。
- `ImagePart` 没有本地路径字段，`aiflect` 不做隐式上传或隐式读取。

### Image Response

Ark image response 到 `ImageGenerationResponse`：

```text
response.data[*].url -> MediaArtifact(kind=image, url=...)
response.data[*].b64_json -> MediaArtifact(kind=image, data=...)
response.data[*].size -> artifact metadata/width/height if parseable
response.usage -> Usage
raw -> raw
```

### Image Stream

Ark image stream event 到 `ImageGenerationStreamEvent`：

```text
image_generation.partial_succeeded -> artifact event
image_generation.partial_failed -> error event
image_generation.partial_image -> partial image data event
image_generation.completed -> completed event with usage
unknown -> raw passthrough
```

### Video Create Request

`VideoGenerationRequest` 到 Ark content generation task：

```text
request.model -> model
request.prompt + input_items -> content
request.duration_seconds -> duration
request.ratio -> ratio
request.resolution -> resolution/provider-native field
request.generate_audio -> generate_audio
request.watermark -> watermark
provider_options -> top-level Ark params
```

Content mapping：

- prompt -> `{"type": "text", "text": ...}`。
- `ImagePart(url=...)` -> `{"type": "image_url", "image_url": {"url": ...}}`。
- `VideoPart(url=...)` -> `{"type": "video_url", "video_url": {"url": ...}}`。
- `AudioPart(url=...)` -> `{"type": "audio_url", "audio_url": {"url": ...}}`。
- `FilePart(url=...)` / `FilePart(data=...)` 需要携带 image/video/audio `media_type` 或 `mime_type`，并映射到对应 URL/base64 media reference。
- `file_id` 与 `local_path` 不隐式上传或读取，当前实现抛 `UnsupportedCapabilityError`。
- role（如 `first_frame`、`last_frame`、`reference_image`）暂不进入 core，可通过 part metadata 或 `provider_options` 显式传递；默认使用 `reference_image`、`reference_video`、`reference_audio`。

### Video Task Response

Ark task 到 `MediaGenerationTask`：

```text
id/task_id -> id
model -> model
status -> TaskStatus
content/output/video url -> MediaArtifact(kind=video)
error -> error
created_at -> created_at
updated_at -> updated_at
usage -> metadata["usage"] or normalized usage if core task later adds usage
raw -> raw
```

## 验收标准

- `OpenAIClient.generate_image()` / `agenerate_image()` 可通过 OpenAI Images API 返回 `ImageGenerationResponse`。
- `OpenAIClient.stream_generate_image()` / `astream_generate_image()` 输出 normalized `ImageGenerationStreamEvent`，不使用 Responses API hosted image generation tool。
- `VolcengineClient.generate_image()` / `agenerate_image()` 可通过 Ark SDK 图片生成返回 `ImageGenerationResponse`。
- `VolcengineClient.stream_generate_image()` / `astream_generate_image()` 输出 normalized `ImageGenerationStreamEvent`，保留 `raw_event`。
- `VolcengineClient.create_video_generation_task()` / async counterpart 可创建视频任务。
- `VolcengineClient.get_video_generation_task()` / async counterpart 可查询并映射任务。
- `VolcengineClient.wait_for_video_generation_task()` / async counterpart 可轮询任务完成、失败或超时。
- Capability 正确声明 OpenAI / Volcengine media generation support。
- Hosted tools 没有稳定 public helper，也不复用 `ToolSpec`。
- 默认 `cd python && ../.venv/bin/python -m pytest` 通过。

## 实现结果

v0.5 已按本方案落地：

- Core 已删除 `ImageGenerationRequest.tools`，并新增/导出 `VideoGenerationRequest`。
- OpenAI adapter 已实现 `generate_image()` / `agenerate_image()` / `stream_generate_image()` / `astream_generate_image()`；无参考图时调用 `images.generate`，存在 `ImagePart(data=...)` 参考图时调用 `images.edit`。
- OpenAI adapter 不使用 Responses API hosted image generation tool；`ImagePart(url=...)` 不会被隐式下载，参考图编辑路径要求显式图片内容。
- Volcengine adapter 已通过 Ark SDK 原生 `images.generate` 实现同步/异步图片生成与图片流式生成。
- Volcengine adapter 已通过 Ark SDK 原生 `content_generation.tasks.create/get` 实现视频生成任务创建、查询和轮询。
- OpenAI 与 Volcengine adapter capability 已声明各自 media generation 能力。
- 默认单元测试不调用真实 provider API。
